По архитектуре:
Для арзитектуры использовался паттерн MVC
Преимущества данного подхода является модульность системы. Бизнес, обработка ввода пользователя и отображения находятся в разных блоках и может изменятся независимо. 
Также преимуществом является распространенность данного подхода.
К недостаткам можно то, что в Console Application модуль View фактический не нужен (собсвтенно, потому весь вывод я и оставил в контроллерах, а на View оставил только саму реализацию вывода).

Общее по сущностям:
Книги были разделены по жанрам: 1) Horrors
				2) Detective
Книги имеют общий интерфейс IBook, базовые функции каждой книги были реализованы в классе Book 

Хранилища книг разделены по наполению: 1) Catalog
				       2) Wastepaper
                                       3) TableStand
                                       4) Sales
Хранилища имеют общий интерфейс IBookStorage, базовый функционал каджого хранилища реализован в классе BookStorage

Для хранения всех моделй используется статический класс ModelStorage (аля БД)

Дополнительно:
Роутинг в каждом контроллере приходилось делать уникальным, т.к вводы в консоли сильно ограничены.
В контроллерах реализован метоп Redirect, который скрывает создание экземпляров контроллера и вызов методов
